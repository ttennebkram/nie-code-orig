searchnames.bat and searchnames.sh Wrappers

SearchNames is written in Java.  Therefore, in order to start SearchNames, it is necessary to start Java.  However, Java takes many command line parameters and can be difficult to start directly.

SearchNames solves his problem by providing a "shell script" that starts the Java executable and passes it the correct parameters to run the SearchNames  application.

On Windows NT, 2000 and XP this script is called searchnames.bat  On Unix distributions, such as Solaris, Linux, FreeBSD, etc. this script is called searchnames.sh

You can pass in command line parameters to these scripts, and they will in turn be passed on to Java and then to the SearchNames application.  For example, your configuration file can be passed in as command line parameter.

See the README file for more details.

"Path Names" in XML

An XML file is a hierarchal tree structure similar to a disk file system.  When developers want to refer to a particular part of an XML tree, they use "path names" that look similar to Unix style directory names.

Some of the SearchNames warning and error messages use this shorthand syntax, so you should be aware of it.

 

Example:

 

In the XML file below, we want to refer to the string "My data"

<xml_level1>

  <xml_level2>

    My data

  </xml_level2>

</xml_level1>

 

This can be represented as:

/xml_level1/xml_level2

 

This conention is a subset of the XML "XPath" syntax.

Overall XML Structure

Main Tags

Main tag is located at /nie_config/search_names

Which would look something like:

<nie_config>

  <search_names …options…>

    ….. search name data ….

  </search_names>

</nie_config>

Critical Options

The main /nie_config/search_names has two important options:

port="nnnn" which port to run on

admin_password="password" the password you will use to control the server

/nie_config/search_names/search_names_url

This is the machine name and port that you will be running SearchNames on.

… header options …

<search_names_url>

  http://my_machine:nnnn

</search_names_url>

Note that you DO need to specify a port in BOTH

<search_names port="nnnn">

AND in

http://my_machine:nnnn

Tell us about your search server.

/nie_config/search_names/fixed_redirection_map

Tell us about search terms you'd like to affect, and what to do when we encounter them.

Notes about <suggestion_marker_text>

This tells us where to look on YOUR results page to put the webmaster suggests

You can have more than one; we'll try them in order.

Notes about <suggestion_marker_text_modifier>

These are options that affect how we match text and where we put our text.

If your marker text contains angle brackets, as HTML OFTEN does, then you should enclose your marker text inside a CDATA section, or use &lt; for <, &gt; for >, etc.

For example, by default we do NOT care about upper vs lower case.  But if you need us to, then you can set that.  Or you may want us to put our suggestion BEFORE your marker text, vs after, or to replace it all together.

Look and Feel Defaults

There are three types of look and feel control you have over the Webmaster Suggests and Alternative Suggestions areas.

ALL of these are set under the path /nie_config/search_names

They can be overridden in specific map tags.  For example, you can have a default icon, and then change it for a few specific terms to something else.

Our apologies for the very rough formatting of this documentation; clearly it needs lots of work.

List of Settings Tags

default_webmaster_suggests_header

Controls the text that says "The Webmaster Suggests".

You can change both the message and the appearance.

Mirrors many of the HTML <font> tag attributes.

Note that this does NOT control the appearance of the titles, summaries and links at this time.

 

default_webmaster_suggests_box

Controls the box around the Webmaster Suggestions.

You can control things like the width, color and border.

Mirrors many of the HTML <table> tag options.

 

default_webmaster_suggests_icon

The icon that is presented in the left part of the Webmaster Suggests box.

If you don't haven an icon, you'll get a green exclamation point.

If you don't want an icon NOR the green exclamation point, then setup an icon that refers to a blank pixel; this will be addressed in a future version.

Mirrors many of the HTML <img> tag options.

 

default_alternate_suggestions_heading

Controls the "Related Terms" message we put before the hyperlinked Alternative Suggetions.

This controls just the intro text, not the links themselves.

Mirrors many of the HTML <font> tag options.

Map Entries

The path /nie_config/search_names/fixed_redirection_map should contain one or more <map> entries.

Each mapping can have more than one search term or phrase that will match.  Also, it can then suggest one or more URL or related term.  The entire map is treated the same, in terms of the action that will be taken, regardless of which term in the group matches.  A term should only be used in one map.

When specifying a URL that will be used as a Webmaster Suggestion, you should also specify a title and description.

Example:

… preamble of config file ….

<fixed_redirection_map>

  <map>

    … map # 1 details …

  </map>

  <map>

    … map # 2 details …

  </map>

  … etc …

</fixed_redirection_map>

Three Basic Types of Map Entries

Sample Map Entry for a Simple Redirect:

<map>

  <term>join</term>

  <term>joining</term>

  <term>join aopa</term>

  <term>joining aopa</term>

  <term>how to join</term>

  <term>how to join aopa</term>

  <term>signup</term>

  <term>sign up</term>

  <term>signing up</term>

  <url>http://www.aopa.org/join/</url>

</map>

Sample Map Entry for a Webmaster Suggests:

<map>

  <term>phil boyer</term>

  <term>boyer</term>

  <term>president</term>

  <url redirect="0">

    http://www.aopa.org/prez/boyerbio.html

    <title>AOPA President Phil Boyer</title>

    <description>

      Phil Boyer became only the third

      full-time president in the 60-year

      history of the Aircraft Owners and

      Pilots Association on January 1, 1991.

      Boyer was previously Senior

      Vice President-Development for Capital

      Cities/ABC Video Enterprises, Inc.

    </description>

  </url>

</map>

Sample Map Entry for an Alternative Suggestion / Related Term:

<map>

  <!-- Map the informal abbrevition for terminal to

    the related correct terms.

    I think Miles had this backwards :-)

  -->

  <term>term</term>

  <alternate_term>terminal</alternate_term>

  <alternate_term>terminal procedures</alternate_term>

</map>

Changing the Wording and Appearance of Map Entries

You can set the options for each map.  If you specified defaults, then setting these options for a particular map will override the default options, within that map.

These options MIRROR the default options in almost every way, except they do not have the "default_" prefix in the tag name.
